# Tauri 更新检测本地测试指南

## 两种测试方式

### 前置步骤：切换配置

测试前需要将更新端点切换到本地服务器：

```bash
# 查看当前配置模式
node scripts/switch-config.js

# 切换到开发测试配置（指向 localhost:3000）
node scripts/switch-config.js dev

# 测试完成后恢复生产配置（指向 GitHub）
node scripts/switch-config.js prod
```

| 模式 | 更新端点 | 何时用 |
|------|----------|--------|
| `dev` | `http://localhost:3000/latest.json` | 本地测试 |
| `prod` | GitHub Releases | 正式发布 |

### 方式一：快速测试（仅检查更新）

测试"检查更新"UI 流程，**不包含下载安装**。

```bash
# 1. 切换到测试配置
node scripts/switch-config.js dev

# 2. 启动本地服务器
node scripts/local-update-server.js

# 3. 打开新终端，运行应用
npm run tauri:dev

# 4. 进入"关于"页面 → 点击"检查更新"
# 5. 应检测到版本 0.99.0

# 6. 测试完成后切回生产配置
node scripts/switch-config.js prod
```

### 方式二：完整测试（含下载安装）

构建真实安装包并签名，测试**完整更新流程**。

```bash
# 1. 首次需生成签名密钥
# tauri signer generate

# 2. 切换到测试配置
node scripts/switch-config.js dev

# 3. 运行完整测试脚本（会自动构建、签名、启动服务器）
node scripts/test-update-full.js

# 4. 安装旧版本的应用 (0.1.0)
# 5. 打开应用 → 关于页面 → 检查更新 → 立即更新
```

## 为什么下载会失败？

Tauri 更新机制要求安装包必须使用私钥签名。虚拟测试文件没有有效签名，因此下载会失败。

```
检查更新（✅ 本地可测）
  │
  └─ 下载更新（❌ 需要真实签名）
       │
       └─ 安装更新（❌ 需要真实签名）
```

- **检查更新**: 只需联网获取 `latest.json`，可本地模拟
- **下载更新**: 需要真实签名的安装包，可用完整测试方案

## 脚本说明

| 脚本 | 用途 |
|------|------|
| `scripts/local-update-server.js` | 快速测试服务器（虚拟安装包） |
| `scripts/test-update-full.js` | 完整测试（构建+签名+服务器） |
| `scripts/switch-config.js` | 切换开发/生产配置 |

## 测试流程图

```
快速测试:                          完整测试:
┌──────────┐                      ┌──────────────┐
│ 启动服务器│                      │ 构建安装包    │
│ (虚拟文件) │                      │ (真实版本)    │
└────┬─────┘                      └──────┬───────┘
     │                                   │
     ▼                                   ▼
┌──────────┐                      ┌──────────────┐
│ 检查更新  │ ✅ 可测              │ 签名安装包    │
└────┬─────┘                      └──────┬───────┘
     │                                   │
     ▼                                   ▼
┌──────────┐                      ┌──────────────┐
│ 下载更新  │ ❌ 签名失败          │ 启动服务器    │
└──────────┘                      └──────┬───────┘
                                         │
                                         ▼
                                  ┌──────────────┐
                                  │ 检查+下载    │ ✅ 完整流程
                                  └──────────────┘
```

## 注意

1. 快速测试仅能验证"检查更新"UI
2. 完整测试需要先生成签名密钥
3. 测试完成后务必切回生产配置：`node scripts/switch-config.js prod`
4. 两个终端需要同时运行：服务器终端 + 应用终端
